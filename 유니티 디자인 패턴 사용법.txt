팩토리 메서드 패턴
종류가 다른 여러 몬스터를 섞어서 생성할 때(생성 패턴)
이벤트 업데이트를 통해 이벤트 몬스터를 생성할때
사용하는 데이터베이스를 변경할때

추상 팩토리 패턴
팩토리 메서드 패턴과 동일하나 상속 구조를 통해 세밀한 팩토리 관리 가능
종족만 다른 몬스터 또는 건물 여러개를 한꺼번에 생성할 때

프로토타입 패턴
비슷하지만 수정이 가능한 기존 오브젝트와 비슷한 오브젝트를 생성할 때 사용

경량 패턴
비슷한 오브젝트를 만들때 동일한 소스가 필요한 부분을 레퍼런스를 통해 한 개의 고유 상태를 공유하여 메모리를 줄임
메시를 공유하는 것이나 스크립트를 addComponent하는 형태로 유니티에서 이미 사용되는 패턴
동일 오브젝트를 생성할 때 하나는 레퍼런스형식으로 받아와서 사용
->오브젝트 풀 패턴에 속한 기능이라고 볼 수 있음

빌더 패턴
비용이 큰 객체를 만들때 전체를 한번에 만들지 않고 부분으로 만드는 기능

상태 패턴
슈퍼마리오 움직임 구현
앉기 + 공격
앉기 + 뒤로
앉기 + 점프는 막음
유한상태기계(fsm)을 사용

옵저버 패턴
주제와 옵저버간의 일대다 형태의 기능을 위해 사용
옵저버가 하는 일은 모두 보고됨, 주제가 공지를 내렸을때 모든 옵저버는 반영됨

어댑터 패턴
완성된 한 클래스의 코드를 변경하지 않고 어댑터 클래스를 사용해 그대로 이용할 수 있게 하는 기능
